# Phase 01 Plan 04: Escalation, Audit Log, and Call Summary Summary

**One-liner:** Warm transfer via Twilio REST API, GDPR audit log writing VoiceCall+VoiceAuditLog+VoiceTranscript on call end, and call summary generated by Claude and saved to QueueUp via PATCH /api/voice/summary with service token auth — completing the full call lifecycle.

## What Was Built

### Task 1: Escalation Detection and Warm Transfer (b8a1ca2)

`voice-service/src/handlers/escalation.ts` provides:
- `ESCALATION_PHRASES`: 10 patient-initiated transfer phrases ("talk to a human", "speak to a receptionist", "real person", "transfer me", etc.)
- `shouldEscalate(transcript, session)`: detects explicit phrase matches OR automatic triggers (unansweredQuestions >= 2, escalationTriggered flag)
- `executeWarmTransfer(callSid, staffPhoneNumber)`: calls `twilioClient.calls(callSid).update()` with TwiML `<Dial>` to redirect the live call to staff — Twilio closes the Media Stream WebSocket automatically after this
- `TRANSFER_BRIDGE_MESSAGE`: bridge message played to patient while connecting ("Of course, I'm connecting you with a team member now...")

### Task 2: Audit Log, Call Summary, and QueueUp API (be9596d)

`voice-service/src/lib/auditLog.ts` provides:
- `writeAuditLog(session)`: creates VoiceCall record (with status 'completed' or 'transferred'), then VoiceAuditLog with all GDPR-03 required fields (consentTimestamp, callSid, clinicId, phoneHash, durationSeconds, actionsJson, wasEscalated), then VoiceTranscript rows with deleteAfter = now + 90 days (GDPR-01 retention)
- `generateCallSummary(session)`: calls Claude sonnet-4-6 with transcript and returns 2-3 sentence summary; returns "Call ended without conversation." for empty sessions
- `saveCallSummary(callSid, summary)`: PATCHes QueueUp API at `/api/voice/summary` with `x-voice-service-token` header

`src/app/api/voice/summary/route.ts`:
- PATCH endpoint validates `x-voice-service-token` against `VOICE_SERVICE_TOKEN` env var
- Returns 401 if token missing or wrong
- Updates `VoiceCall.summary` field via `prisma.voiceCall.updateMany({ where: { callSid } })`
- Returns 404 if VoiceCall not found

`voice-service/src/handlers/twilioStream.ts` (wired):
- `start` event: plays consent greeting via ElevenLabs TTS
- `transcript` handler: consent gate -> escalation check -> Claude conversation
- `stop` event: adds 'call_ended' to actionsLog, calls writeAuditLog -> generateCallSummary -> saveCallSummary (fire-and-forget with error logging)

`prisma/schema.prisma`: Added `summary String?` to VoiceCall model, pushed to DB, regenerated client.
