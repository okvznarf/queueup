    **chat-widget/src/session.ts:**
    - `const SESSION_KEY_PREFIX = 'queueup_session_';`
    - `export function getSessionId(shopId: string): string` — reads `sessionStorage.getItem(SESSION_KEY_PREFIX + shopId)`. If null, generates `crypto.randomUUID()`, stores it, returns it.
    - `export function getMessages(shopId: string): Array<{role: string; content: string}>` — reads `sessionStorage.getItem(SESSION_KEY_PREFIX + shopId + '_messages')`, parses JSON, returns `[]` if null/error.
    - `export function saveMessages(shopId: string, messages: Array<{role: string; content: string}>): void` — writes to sessionStorage with the shopId-namespaced key.
    - `export function clearSession(shopId: string): void` — removes both keys.

    **chat-widget/src/api.ts:**
    - `export async function sendMessage(apiUrl: string, shopId: string, sessionId: string, message: string, onChunk: (text: string) => void): Promise<void>`
    - Sends `fetch(apiUrl + '/chat', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ shopId, sessionId, message, consentGranted: true }) })`.
    - Reads response body as ReadableStream. Decodes SSE events line by line.
    - For each `data: {...}` line, parse JSON. If `type === 'text'`, call `onChunk(parsed.delta)`. If `type === 'done'`, return.
    - Handle fetch errors gracefully: if response not ok, throw `new Error('Chat request failed: ' + response.status)`.

    - `export function MessageList({ messages }: { messages: Array<{role: string; content: string}> })`
    - Maps over messages, renders `<div class={msg.role === 'user' ? 'queueup-msg-user' : 'queueup-msg-ai'}>{msg.content}</div>`.

    **chat-widget/src/App.tsx:**
    - `import { useState, useEffect, useRef } from 'preact/hooks';`
    - Props: `{ shopId: string }`
    - State: `isOpen` (boolean, false), `consented` (boolean, false), `messages` (array), `input` (string), `isLoading` (boolean), `shopConfig` ({ shopName, primaryColor } | null), `streamingContent` (string, for in-progress AI response).
    - On mount (`useEffect([], ...)`):
      1. Fetch `/api/widget/config?shopId=${shopId}` from `window.QUEUEUP_API_URL || window.location.origin` — store in `shopConfig`.
      2. Load messages from `getMessages(shopId)` — restore prior conversation.
      3. Check if consent was previously granted: `sessionStorage.getItem('queueup_session_' + shopId + '_consent') === 'true'` — if so, set `consented = true`.
    - `handleAccept`: set `consented = true`, store `sessionStorage.setItem('queueup_session_' + shopId + '_consent', 'true')`.
    - `handleDecline`: set `isOpen = false` (close panel).
    - `handleSend`:
      1. If `!input.trim() || isLoading` return.
      2. Add `{ role: 'user', content: input }` to messages. Clear input. Set `isLoading = true`. Set `streamingContent = ''`.
      3. Get sessionId from `getSessionId(shopId)`.
      4. Determine API URL: `window.QUEUEUP_CHAT_API_URL || 'https://voice.queueup.com'` (configurable for dev).
      5. Call `sendMessage(apiUrl, shopId, sessionId, input, (chunk) => { streamingContent += chunk; })`.
      6. After complete: add `{ role: 'assistant', content: streamingContent }` to messages. Set `isLoading = false`. Clear `streamingContent`.
      7. Call `saveMessages(shopId, messages)` to persist.
      8. On error: add `{ role: 'assistant', content: 'Sorry, something went wrong. Please try again.' }` to messages. Set `isLoading = false`.
    - Render:
      - If `!shopConfig` → render nothing (still loading config).
      - Always render bubble: `<button class="queueup-bubble" onClick={() => setIsOpen(!isOpen)}>` with a chat icon (SVG or text).
      - If `isOpen`:
        - Render panel with header showing `shopConfig.shopName` and close button.
        - If `!consented` → render `<Consent onAccept={handleAccept} onDecline={handleDecline} />`.
        - If `consented` → render `<MessageList messages={messages} />`, show `streamingContent` as partial AI message while loading, input area with text field and send button.

    **chat-widget/src/widget.tsx:**
    - Entry point. Self-executing mount function.
    - `function mount()`:
      1. Find current script tag: `document.currentScript as HTMLScriptElement` (works because script is loaded with `async` not `defer`). Fallback: `document.querySelector('script[data-shop-id]')`.
      2. Read `shopId` from `script.dataset.shopId || script.getAttribute('data-shop-id')`.
      3. If no shopId, `console.error('QueueUp widget: data-shop-id attribute required'); return;`
      4. Optionally read `data-api-url` for API base URL override: `(window as any).QUEUEUP_CHAT_API_URL = script.dataset.apiUrl || (window as any).QUEUEUP_CHAT_API_URL;`
      5. Optionally read `data-app-url` for Next.js base URL override: `(window as any).QUEUEUP_API_URL = script.dataset.appUrl || (window as any).QUEUEUP_API_URL;`
      6. Create host div: `const host = document.createElement('div'); host.id = 'queueup-widget-host'; document.body.appendChild(host);`
      7. Attach shadow DOM: `const shadow = host.attachShadow({ mode: 'open' });`
      8. Fetch config to get primaryColor (or use default #6366f1), inject styles: `const style = document.createElement('style'); style.textContent = getStyles(primaryColor); shadow.appendChild(style);`
      9. Create mount point: `const root = document.createElement('div'); shadow.appendChild(root);`
      10. `render(<App shopId={shopId} />, root);`
    - Call `mount()` at module level (esbuild IIFE executes immediately).
    - Note: `document.currentScript` is null after DOMContentLoaded in some browsers. Since the script is `async`, it runs during parsing when `document.currentScript` is available. As a fallback, query `script[data-shop-id]`.
